methods of proof

If you’re completely new to maths, and you don’t get how proofs work (and you wanna learn how they work), then this is the post for you.

What is a proof in mathematics? Think of a proof as nothing more than an argument that convinces people that you’re right. But not just any argument — one that cannot be questioned.

For instance, consider the following question:

Prove that n2+2n2+2 is never a perfect square when nn is a non-negative integer.

It might look a bit daunting. You could say something like “I tried for a bit but I couldn’t find any which were squares”. Definitely not a proof, as it’s hard for that to convince anybody that you’re correct.

“Well n2n2 is a square and that’s two away so it’s probably not a square either because square numbers are really far apart” comes closer, but obviously “square numbers are really far apart” is not accurate. For instance, 0 and 1 are only one number apart.

A mathematical proof would build on that idea as follows:

If n=0n=0, then n2+2=2n2+2=2 which is not a perfect square.
If n≠0n≠0, then

n2<n2+2<n2+2n+1=(n+1)2n2<n2+2<n2+2n+1=(n+1)2
Now if you think about what this is telling you, it’s saying that n2+2n2+2 lies between two consecutive perfect squares. But that means n2+2n2+2 can’t itself be a perfect square, because perfect squares never lie between two other consecutive perfect squares (by definition of perfect square).

Don’t worry if you didn’t get that proof. Read through the post below and come back and see if it makes sense then!

How exactly do you prove stuff? Here are some methods:

Direct proof

This means you directly prove something. You start at the start, and via a chain of deductions, you arrive at the finish. For instance, the above proof is a direct proof. Another example is as follows:

Question: prove that for all reals aa and bb,

a2+b2≥2aba2+b2≥2ab

Proof: Note that x2≥0x2≥0 for all x∈Rx∈R. The proof of this is trivial and left as an exercise for the reader (but if you really can’t get it, consider what happens if xx is positive, negative, and 00).

Then letting x=a−bx=a−b, we know that a2−2ab+b2≥0a2−2ab+b2≥0 and rearranging this proves the question.

Direct proof also has a subtopic known as “divide and conquer” or “case bash”: what you do is you divide the problem up into a bunch of subproblems, and prove that each subproblem is true. I’ve already used this twice in this post, but another example can’t hurt:

Question: prove that for any positive integer nn, one of n,n+1n,n+1 and n+2n+2 is divisible by 33.

Proof: If you remember division from primary school, you’ll know that any number divided by 33 leaves a remainder which is either 00 (no remainder), 11 or 22. If the remainder is 33 or more then we divided incorrectly. We consider what happens when we divide nn by 33:

If nn leaves a remainder of 00, then nn is, by definition, a multiple of 33 (n=3kn=3k is a multiple of 33)

If nn leaves a remainder of 11, then n+2n+2 leaves a remainder of 00 and is thus a multiple of 33 (n=3k+1n=3k+1 so n+2=3(k+1)n+2=3(k+1) is a multiple of 33)

If nn leaves a remainder of 22, then n+1n+1 is divisible by 33 (see if you can write out your own bracket for this case!)

Proof by contradiction

“If I’d died, I’d be in hell”

Proof by contradiction works by assuming the opposite of the question is true and proving that it ends up being absurd. Let’s look at an example.

Question: prove that if n2n2 is even then so is nn.

Proof: Given that n2n2 is even, suppose nn is odd. As we all know, an odd number times an odd number is odd so n×n=n2n×n=n2 is odd. But that’s a contradiction! We supposed that n2n2 was even at the start. Thus somewhere along the line, we made an incorrect deduction, and the only place that could’ve happened was when we said nn is odd. Hence nn must be even.

Let’s do some formal logic before moving onto the next one.

A⟹BA⟹B tells us that if AA is true, then BB is true, and is read “A implies B”.

Let’s look at an example. If A is the statement “The year is 2019” and B is the statement “It’s the 21st century” then A⟹BA⟹B is a true deduction: if A (the year is 2019) is true, then B (it’s the 21st century) is true as well. On the other hand, B implies A is false: just because it’s the 21st century doesn’t mean the year is 2019. It could be 2018 or any of another 99 years.

One more example? Sure — A is the statement “In a triangle, a2+b2=c2a2+b2=c2 where cc is the longest side” and B is the statement “The triangle is right-angled”. We know by pythagoras that B⟹AB⟹A — that is, if a triangle is right angled then a2+b2=c2a2+b2=c2. But does A⟹BA⟹B hold? In fact it does also — if a2+b2=c2a2+b2=c2, then the triangle is right-angled.

Proof by contrapositive

This is really just proof by contradiction, but renamed and revamped.

If we let ¬¬ mean “not” (for instance, if A is the statement “a is positive” then ¬A¬A means “a is not positive”), then

A⟹BA⟹B is true if and only if ¬B⟹¬A¬B⟹¬A is true.

Wait a second. “if and only if”? What does that mean?

If and only if (commonly abbreviated iff) is a mathematical shorthand for saying that two things are equal. A iff B is just saying that A⟹BA⟹B and B⟹AB⟹A, that is, if any one is true then both of them are true. For instance, in the second example, a triangle is right angled iff a2+b2=c2a2+b2=c2. In the first example, however, iff doesn’t hold: B⟹AB⟹A isn’t true, as we saw.

Anyways, what all this is saying is that if a question asks you to prove that if AA is true, then BB is true, you can do the opposite: prove that if BB is not true, then prove that AA isn’t either.

Let’s revisit one of our examples.

Question: prove that if n2n2 is even then so is nn.

Proof: We prove the contrapositive: if nn is not even, then n2n2 is not even either. But this is obvious — if nn is odd, then obviously n2n2 is odd too, so we’re done.

Note how reminiscent of contradiction this is. It’s pretty much the same thing.

One other thing that’s always in “methods of proof”:

induction

Suppose you wanna prove that something is true for all positive integers nn. If you can prove that it’s true for 1, and that if it’s true for k then it’s true for k+1, then what you do is set off a domino effect: it’s true for 1, so it must be true for 2, so it must be true for 3, so it must be…

This means that you’re done!

Okay, that probably didn’t make much sense to people who don’t already know induction. Time for an example.

Question: prove

(1+2+3+⋯+n)2=13+23+33+⋯+n3(1+2+3+⋯+n)2=13+23+33+⋯+n3

Proof: (base case) When n=1n=1, obviously 12=1312=13.

(inductive step) Suppose (1+2+3+⋯+k)2=13+23+33+⋯+k3(1+2+3+⋯+k)2=13+23+33+⋯+k3. Then

====13+23+33+⋯+k3+(k+1)3(1+2+3+⋯+k)2+(k+1)3(1+2+3+⋯+k)2+(k+1)2+k(k+1)2(1+2+3+⋯+k)2+(k+1)2+2(k+1)(1+2+3+⋯+k)(1+2+3+⋯+k+(k+1))213+23+33+⋯+k3+(k+1)3=(1+2+3+⋯+k)2+(k+1)3=(1+2+3+⋯+k)2+(k+1)2+k(k+1)2=(1+2+3+⋯+k)2+(k+1)2+2(k+1)(1+2+3+⋯+k)=(1+2+3+⋯+k+(k+1))2

Let’s just trust the algebra for now, and forevermore.

Anyways, what does this tell us? We know that the problem statement is true for n=1n=1. But that means that it’s true for n=2n=2 (when k=1 in the inductive step). But then it’s true for n=3n=3 (k=2 in the inductive step). Therefore it’s also true for n=4n=4 (k=3) and so on and so on. Thus it’s true for all nn so we’re done.

If you don’t get that, don’t worry. It’s not that important, but if you have to learn it, and it still doesn’t make sense, then look it up online elsewhere I guess? I mean, most questions are doable without this technique.
