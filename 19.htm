diophantine equation tactics

WARNING! This post contains some pretty tricky maths. If you read the title and thought ‘ahh, good old diophantine equations’ then this post is for you. Otherwise, well, feel free to give reading this a shot anyway 🙂

I guess I need to write an introduction for this topic. Diophantine equations are mathematical equations where the variables must be solved over integers, or positive integers, or something like that. Don’t worry if that didn’t make sense, have some examples and see if you can pick it up. Oh, and unless otherwise stated, we’ll be working in positive integers.

Protip: try each question before reading the solution. Hover over the thingies to read the solution.

factorise | Difficulty: 1*

Find all x,yx,y such that 4xy−x+y=2019.4xy−x+y=2019.

This is the same as (4x+1)(4y−1)=8075=52⋅17⋅19(4x+1)(4y−1)=8075=52⋅17⋅19 which trivially resolves to (x,y)=(1,404),(4,119),(6,81),(21,24),(106,5)(x,y)=(1,404),(4,119),(6,81),(21,24),(106,5).

take out gcd | 1.5*

Solve a2+b2=c2.a2+b2=c2.

Write d=gcd(a,b,c)d=gcd(a,b,c). Then we can write a=dA,b=dB,c=dCa=dA,b=dB,c=dC. Consider the equation mod 4, and you can WLOG that AA is even, and BB and CC are odd. Thus A2=C2−B2A2=C2−B2 so A2=(C+B)(C−B)A2=(C+B)(C−B). Thus each of C+BC+B and C−BC−B are squares, or twice squares. They can’t both be even squares, because they have a gcd of 1, so therefore C+B=2u2C+B=2u2, C−B=2v2C−B=2v2 for some u,vu,v which gives a parametrisation (a,b,c)=(2duv,du2−dv2,du2+dv2)(a,b,c)=(2duv,du2−dv2,du2+dv2) for gcd(u,v)=1gcd(u,v)=1 and u,vu,v not both odd. And you can swap aa and bb because of the WLOG.

This is pretty basic so far. Let’s up the ante, shall we? protip: that also means solution quality decreases. Oh well, I’m happy to pay that price.

infinite descent | 1.5*

Find all integer solutions to x3+3y3+9z3=0.x3+3y3+9z3=0.

Assume you have a solution, (x,y,z)(x,y,z). Then, since 3 is a factor of xx, we can reduce the equation down. Then we notice that 33 is a factor of yy as well, and hence is a factor of zz. These factors can be removed ad infinitum, which means that (0,0,0)(0,0,0) is the only solution. Pretty magic, huh?

bounding arguments | 2*

Find all positive integers such that 1a+1b+1c+1d=1.1a+1b+1c+1d=1.

This one isn’t too hard, WLOG a≤b≤c≤da≤b≤c≤d and then bash. Obviously a=2,3,4a=2,3,4 (one of them).

Then bound bb in the same way, for each case, and finally work out which cases actually work. Easier done than said.

Apparently there are 215 solutions. Took me a page (albeit with very large writing and lots of wasted space).

mods | 2*

Show 2x+2+3y=7z2x+2+3y=7z has no solutions.

Consider the equation (mod 3), and you get that x=2ax=2a for some aa. If you consider the equation (mod 8), you notice that yy and zz have to be even. Let z=2cz=2c. Thus, we can conclude 7b−2a+1,7b+2a+17b−2a+1,7b+2a+1 are both powers of 33, which is obviously not doable (mod 3), or 7c−2a+1=17c−2a+1=1, which fails (mod 7).

vieta jumping | 3*

If kab=a2+b2+1kab=a2+b2+1, prove k=3.k=3.

Thanks, wikipedia.

Firstly WLOG a>ba>b; the equality case is trivial.

The general gist is that we can write aa as a root of the quadratic x2−kbx+(b2+1)=0x2−kbx+(b2+1)=0. Thus the other root must be a′=kb−a=b2+1aa′=kb−a=b2+1a. The other root is positive and integral (since a′+a=kba′+a=kb), and if b>1b>1 then a′<ba′<b (since a′a=b2+1a′a=b2+1).

Since the question is symmetric in aa and bb, we can therefore generate b′b′ in a similar manner, until we hit b=1b=1 in which case k=3k=3 is forced. Since kk doesn’t change the whole way, we conclude that k=3k=3 no matter what.

cyclotomic polynomials | 4*

Solve x2=y7+7.x2=y7+7.

x2+121=y7+128x2+121=y7+128. This means that y≡1y≡1 mod 4. Now, consider a prime p≠11p≠11 which is a factor of the LHS and show that p≡1p≡1 mod 4, via cyclotomics or otherwise. Factorise the RHS and show that 1111 must therefore divide both parts. Thus x=11ax=11a so 11 divides each of the factors on the RHS. This means that y≡9y≡9 mod 11 but that kills the sextic factor.

unique factorisation domain | 5*

Solve x2+2=y3.x2+2=y3.

So this thing requires considering Z[−2−−−√]Z[−2] and showing that it is a unique factorisation domain, and then showing that x+−2−−−√=(a+b−2−−−√)3x+−2=(a+b−2)3 by considering gcds. From there it’s just a matter of expanding and solving. Not hard at all.

Yeah good luck. You only have Euler to blame.
