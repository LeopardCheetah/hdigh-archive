integer functional equations

(NN doesn’t include 0 in this post. Sorry for all the people in that faction.)

Functional equations involving integers are susceptible to a few main tricks, and, of course, all of the regular tricks for functional equations.

Oh, but wait — am I getting ahead of myself? Ah — of course! I forgot to explain what an integer functional equation was.

Basically a function that takes in integers, or some subset thereof, and outputs integers. Pretty basic stuff. If you don’t know what a function is then I suggest reading the related post.

Anyways, onto tricks.

Induction is the first obvious one. If you can work out f(1)f(1) and that forces f(2)f(2) which forces f(3)f(3) and so on then you’re done. If you recall, this is how you solve Cauchy’s over the integers.

Another useful thing is the fact that the naturals are well-ordered, and the set of values your function can take must have a minimum. For example, take the following:

Find all functions f:N→Nf:N→N such that f(n)=f(2n)+f(3n)f(n)=f(2n)+f(3n) for all n∈Nn∈N.

Consider the smallest value ff takes (call it aa), and choose an nn that gives it this value. Then a=f(n)=f(2n)+f(3n)≥2aa=f(n)=f(2n)+f(3n)≥2a, which means a≥2aa≥2a, and this is impossible. So no such functions exist.

A large number of integer functional equations are primarily number theoretic. In these cases, considering prime factors of inputs/outputs can often be useful. This is especially useful if you have functional equations of the form (some stuff with f) divides (other stuff with f); if you can get the RHS to be a prime, then the left hand side is either 1, or equal to the RHS (or negatives of those two, if that’s possible). Even if primes aren’t always possible, setting the RHS to something with a limited number of possible factors will greatly narrow down the possibilities that the LHS can take.

Another important thing to remember is that a∣ba∣b implies a≤ba≤b, and so if you can possibly fix the RHS and make the LHS arbitrarily large you can derive a contradiction.

Let’s look at another example.

Just a note, to you experienced veterans out there, these examples are actually frighteningly difficult. Give them a shot if you want a bit of a tasty challenge without getting over-the-top-imo-q3/6-level-difficult.

Find all functions f:N→Nf:N→N such that

f(a+b)+b∣a+2f(b)f(a+b)+b∣a+2f(b)
for all naturals a,ba,b.

Let’s see… how do we make a divide symbol actually useful? A divides symbol doesn’t really seem to help much. If only we could get rid of it– but of course we can! If we can make the RHS a prime, then both sides have to be equal. Thus setting a+2f(b)=pa+2f(b)=p for some large pp gives us that

f(p−2f(b)+b)+b=pf(p−2f(b)+b)+b=p
for all sufficiently large primes pp.

Notice that what this tells us is that f(p−c)=p−bf(p−c)=p−b for all primes pp and some fixed integer cc. In particular, this means that there exists very large nn such that f(n)∼nf(n)∼n where ∼∼ means “is very close to” in some precise sense we’ll get to later.

For fixed b1,b2b1,b2, consider two very large a1,a2a1,a2 (relative to bi,f(bi)bi,f(bi)) such that ai+biai+bi are the same such nn. Since we know that both sides are roughly the same number, we know that in fact they must both be equal (that’s how divisibility works). Thus

f(n)+bi=(n−bi)+2f(bi)f(n)+bi=(n−bi)+2f(bi)
Subtracting the two equations and varying the fixed bibis (I know, preposterous), we get that f(n)=n+cf(n)=n+c for all nn so ff must in fact be constant.

One last example.

Find all f:N→Nf:N→N such that for all positive integers a,ba,b,

f(a)+b∣a2−f(b2)f(a)+b∣a2−f(b2)

This question is a little ridiculous in the sense that if you try it using the previous technique, it just doesn’t work. The RHS is too sturdy to force into a prime or something (as you can probably tell if you’ve guessed the solution already).

For this question, it helps, as almost always, to try simple values. Let’s look at a=b=1a=b=1:

f(1)+1∣1−f(1)f(1)+1∣1−f(1)
which quickly gives f(1)=1f(1)=1. Nice start. Now that we know 11, let’s put b→1b→1: f(a)+1∣a2−1f(a)+1∣a2−1. Nice, nice. If a=2a=2 this gives us that f(2)=2f(2)=2. Wouldn’t it be nice knowing f(4)f(4)? Let’s try b=2b=2 and a=1a=1:
3∣1−f(4)3∣1−f(4)
Interesting. And we also know that
f(4)+1∣15f(4)+1∣15
which thus tells us that f(4)=4f(4)=4. Nice stuff.

Now let’s try some good ol’ fashioned induction. Suppose f(n)=nf(n)=n and f(n2)=n2f(n2)=n2 for all n≤kn≤k. Then for k+1k+1, setting a=k+1a=k+1 and b=kb=k gives f(k+1)+k∣2k+1f(k+1)+k∣2k+1 so f(k+1)=k+1f(k+1)=k+1. Finally, consider what happens when we let b=k+1b=k+1, and let aa vary. We have a+b∣b2−f(b2)a+b∣b2−f(b2) (yes, that’s what I mean; can you see why?) for all a≤k+1a≤k+1.

We also know that f(b2)+a∣b4−a2≡b4−f(b2)2=(b2+f(b2))(b2−f(b2))f(b2)+a∣b4−a2≡b4−f(b2)2=(b2+f(b2))(b2−f(b2))

The problem isn’t too hard from here; assume f(b2)≠b2f(b2)≠b2 and bound it from above as well as from below. It’s just really messy and above my paygrade to latex it out for you. Sorry! You might also want to manually work out a few small values of ff first too (in the same way that f(1)f(1) and f(2)f(2) were evaluated), but it’s possible without more manual labour.
