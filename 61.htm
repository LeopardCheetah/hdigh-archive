functional equations (easy)

Personally, I find solving functional equations to be one of the funnest things in olympiad mathematics.

I’ll just add something: I’m the type of person that likes going on tangents (yeah this is meta (even these brackets are meta lol)) so if I add a random tangential comment that you don’t understand, don’t worry, it’s probably just a random example that isn’t actually relevant.

Also just a note to experienced veterans, if you want just skip the first section. You’re not missing out on much, just some fluffy analogies and some wry humour.

what is a function

But let’s go back a step first. What is a function? What is a functional equation? What do I need to know? How do you solve one? What are we having for dinner tonight? Why is the sky blue? So many questions!

A function f:A→Bf:A→B is a subset of S⊂A×BS⊂A×B such that

∀a∈A,|{(a,b)∈S}|=1∀a∈A,|{(a,b)∈S}|=1

Think of a function as a mysterious machine. It’s got input and output chutes. You put stuff in the input, and stuff comes out the output.

But these machines can’t just do anything. There are certain rules and stuff they have to follow. Here we go:

  • THE IMPORTANT ONE If you put the same thing in, the same thing comes out
  • THE ONE PEOPLE FORGET THE MOST OFTEN You can’t just put anything in; there’s a ruleboard that explains which things can go in and which things can’t
  • hi i exist too It can’t just spit anything out; there’s another ruleboard that tells you what stuff could possibly come out

Hey, that wasn’t too bad. Let’s jump straight to numeric functions (although functions don’t have to only have numbers; for instance, differentiation is a function that sends differentiable functions to functions, and “number of sides” is a function that sends polygons to positive integers).

So an example of a function for positive integers would be the “same number” function: whatever you put into the function is what it spits back out. Sounds pretty lame. Let’s give this function a name: let’s call it ss for same. Since mathematicians are lazy people, we’ll use some notation for this: when we apply ss to a number, we’ll write it as s(random number)s(random number).

There. That should make life easier, and definitely not more confusing, because “normally” s(t+2)s(t+2) means ss times t+2t+2. But remember! In this case there is no multiplication going on. You’ll get used to it.

functional equations

Let’s try a functional equation now.

A functional equation is an equation where… I think an example is worth a thousand words. Here, let’s start with a problem I saw earlier today:

Find all f:R→Rf:R→R such that for all real x,yx,y,

f(x2+f(y))=y+f(x)2f(x2+f(y))=y+f(x)2

First and foremost: substitution. An IMO gold medalist once told me that the only way to solve functional equations was to sub sub sub sub sub sub… (if I recall correctly, an entire page of the handout was the word “sub” printed over and over).

What is substitution? See how at the top it says that the equation is true for all real x,yx,y? We just replace these xx and yy with whatever we want. For instance, if xx and yy were both 00 then the equation simply tells us that f(f(0))=f(0)2f(f(0))=f(0)2.

There are two “well-known” tricks to functions, and these are known as injectivity and surjectivity. Not every function is injective or surjective! It’s important to remember that some functions are neither, and some are both.

Injectivity tells you that when you put in different things into a function, the outputs will be different.

Surjectivity tells you that at some point, everything in the “all possible outputs” will be hit.

For instance, k:Z→Zk:Z→Z where k(x)=x2k(x)=x2 is neither injective nor surjective, but l:Z→Zl:Z→Z where l(x)=xl(x)=x is both (known as bijective).

How do you prove either injectivity or surjectivity? Let’s take a look:

Suppose that two numbers give the same output, that is, f(a)=f(b)f(a)=f(b). Then

f(x2+f(a))=a+f(x)2f(x2+f(a))=a+f(x)2
and
f(x2+f(b))=b+f(x)2f(x2+f(b))=b+f(x)2

But wait! The LHSs (left hand sides) of the equations are the same (because f(a)=f(b)f(a)=f(b)) so we know that the RHSs are the same, and since same inputs give same outputs, we know that a=ba=b. Thus different inputs must give different outputs, so this function is injective.

On the other hand, if we look at the left hand side, we notice that it’s f(stuff)f(stuff). On the other hand, the RHS is y+f(x)2y+f(x)2. If we set xx to some constant and vary yy, notice that the RHS can attain any real value, and thus so can the left hand side. But if f(stuff)f(stuff) can attain any real value, by definition ff is surjective!

Now for some functional magic. We know by surjectivity that there exists cc such that f(c)=0f(c)=0. Thus

y+f(−c)2=f(c2+f(y))=y+f(c)2=yy+f(−c)2=f(c2+f(y))=y+f(c)2=y
and so f(−c)=0f(−c)=0. But wait! Because ff is injective, we know that f(c)≠f(−c)f(c)≠f(−c) unless c=0c=0. So by some functional manipulation, we get that f(0)=0f(0)=0.

Let’s use this fact! If we put x→0x→0 in the function we get that f(f(y))=yf(f(y))=y. Similar, y→0y→0 gives f(x2)=f(x)2f(x2)=f(x)2. Thus we can put y=f(b),x=ay=f(b),x=a and rewrite the equation as:

f(a2+b)=f(a2)+f(b)f(a2+b)=f(a2)+f(b)

Wouldn’t it to be nice to get rid of those square symbols and just turn it into f(x+y)=f(x)+f(y)f(x+y)=f(x)+f(y)? This is actually doable, and while we’re here let’s just prove that this fact is true: since f(x2)=f(x)2f(x2)=f(x)2, we know that f(x)2=f(−x)2f(x)2=f(−x)2 and hence f(x)=±f(−x)f(x)=±f(−x). But since ff is injective, they are the same sign only if x=0x=0, and so f(x)=−f(−x)f(x)=−f(−x) (known as an “odd” function) (note that this is still true when x=0x=0)

Let’s do some cases: if xx or yy is non-negative, WLOG xx is, and write x=a2x=a2 and we get the desired equality.

Otherwise, xx and yy are negative, and so f(−x−y)=−f(x)−f(y)f(−x−y)=−f(x)−f(y) is equivalent to what we want to prove. This is true by setting −x=a2−x=a2 and y=−by=−b.

Also, apologies for the heavy variable reuse. It’s usually what happens in functions; you just remember which variable is getting reused where. Bad practise but commonplace, much like… actually I won’t make that joke.

Anyways, f(x+y)=f(x)+f(y)f(x+y)=f(x)+f(y) is what we have. This guy is known as Cauchy’s functional equation and the solution is pretty magic. I’ll skim the details.

Obviously f(0)=0f(0)=0. Assume f(1)=cf(1)=c and induct to get that f(n)=ncf(n)=nc for all natural numbers nn. Prove then that f(nr)=nf(r)f(nr)=nf(r) for all naturals nn and reals rr and deduce that f(qpq)=qf(pq)f(qpq)=qf(pq) and thus that f(q)=qcf(q)=qc for all rationals qq.

The trouble is, you can’t actually solve Cauchy’s on the reals. It’s just impossible — there are too many functions. Fortunately, if you know that Cauchy’s is bounded (above or below) on some interval, then it’s solvable, and f(x)=cxf(x)=cx for all xx. This is one of the exercise below because I’m too lazy to write it out.

And hey presto, f(x2)=f(x)2f(x2)=f(x)2 so ff is non-negative for non-negative inputs so ff is indeed bounded below and we’re done. Thus f(x)=cxf(x)=cx. Substituting this back into the original equation gives

cx2+c2y=c2x2+ycx2+c2y=c2x2+y
and thus c=1c=1 giving us f(x)=xf(x)=x, the boring old identity function, as the only solution.
